## Choosing a Signer

When selecting a signer, consider:

1. **Security Requirements**: Hardware security keys and biometrics offer enhanced security
2. **User Experience**: Social logins provide familiar authentication flows
3. **Integration Complexity**: Some signers require additional setup and configuration
4. **Enterprise Features**: Custody solutions offer advanced key management
5. **Cost**: Different providers have varying pricing models

## Implementation

Each signer implementation follows a similar pattern:

```typescript
import { type Signer } from "permissionless"

// Initialize the signer
const signer = await createSigner({
  // Signer-specific configuration
})

// Use the signer with a smart account
const account = await createSmartAccountClient({
  signer,
  // Other account configuration
})
```

For specific implementation details, refer to the individual signer documentation pages listed above.
